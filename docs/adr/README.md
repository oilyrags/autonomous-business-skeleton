# Architecture Decision Records (ADRs)

Single-context layout: ADRs for the whole project live here as `NNNN-title.md`.

These are created **lazily** — the `/domain-modeling` skill (reached via `/grill-with-docs` and `/improve-codebase-architecture`) writes an ADR when a hard-to-explain decision actually gets resolved. Don't pre-author ADRs; let them accrue as decisions are made.

Format: see [`.claude/skills/domain-modeling/ADR-FORMAT.md`](../../.claude/skills/domain-modeling/ADR-FORMAT.md).

> Note: lightweight *architecture* decisions captured during design also appear in the v1.0 package under [`architecture/`](../../architecture/) and the project decision log in [`PROJECT.md`](../../PROJECT.md). ADRs here are the granular, code-level records going forward.
